##### Zusammenfassung 7

##### Notizen 7

Es gibt verschiedenen Wege Geräte von Endbenutzer de**m** **Intune** **Endpoint Management (IEM) [hinzugefügt](https://www.dwds.de/wb/hinzuf%C3%BCgen)** werden können.

In diese Folge, geht es darum, dass Geräte, die bereits Bestandteile unsere lokale **Active Directory (LAD)** Umgebung sind, auch Bestandteile userer **Intune Endpoint Management (IEM)** Umgebung werden, [insbesondere](https://www.collinsdictionary.com/dictionary/german-english/insbesondere) wird es gezeigt wie ein **Hybrid Azure AD Joined** Device (**HADJ**) mit **IEM** integriert werden kann, [**dazu**](https://www.collinsdictionary.com/dictionary/german-english/dazu) wird eine **Gruppen[richtlinie](https://www.dwds.de/wb/Richtlinie) für** eine [Organisation](https://www.collinsdictionary.com/dictionary/german-english/organisation)s[**einheit**](https://www.collinsdictionary.com/dictionary/german-english/einheit) innerhalb die lokale Active Directory Umgebung [**eingerichtet**](https://www.collinsdictionary.com/dictionary/german-english/einrichten)**.**   
   
Die **Gruppen[richtlinie](https://www.dwds.de/wb/Richtlinie)** wird mittels \[ **Server Manager (SM)** \> Tools \> **Gruppenrichtlinienverwaltung** \] **auf** de**m** lokale**n** **Active Directory Domain Controller** (**AD-DM**) **eingerichtet** und [konven**t**ionell](https://www.collinsdictionary.com/dictionary/german-english/konventionell) **MDM Auto-Enrolment** genannt, jedoch kann man diese [beliebig](https://www.collinsdictionary.com/dictionary/german-english/beliebig) nennen, dann wird die Richtlinie zu eine [Organisation](https://www.collinsdictionary.com/dictionary/german-english/organisation)s[**einheit**](https://www.collinsdictionary.com/dictionary/german-english/einheit) [innerhalb](https://www.collinsdictionary.com/dictionary/german-english/innerhalb) de**s AD-DM** zugewiesen.

Der nächste Schritt ist die **Gruppen[richtlinie](https://www.dwds.de/wb/Richtlinie)** zu gestalten so dass, sie die **Automatische MDM Registrierung** unterstützen wird. Man muss dies**e Option** manuell **[aktivieren](https://www.collinsdictionary.com/dictionary/german-english/aktivieren)**, dazu wird man auf de**r** Richtilinie**object** mit dem rechten Knopf **klicken**, um sie zu **bearbeiten**. 

| Auf dem lokalen Active Directory Domain Controller (AD-DM) Richtlinien \> Admistrative Vorlagen \> Windows Komponenten \> MDM \> Automatische MDM Registrierung unterstützen wird. Man muss diese Option [aktivieren](https://www.collinsdictionary.com/dictionary/german-english/aktivieren)  Benutzeranmeldedaten auswählen, dazu der Benutzer mit seinen eigenen Microsoft 356 Benutzername anmelden kann. [Schließlich](https://www.collinsdictionary.com/dictionary/german-english/schliesslich) wird man die neue Group Policy (GP) mit dem [Befehl](https://www.collinsdictionary.com/dictionary/german-english/befehl) [gpupdate](https://learn.microsoft.com/en-us/windows-server/administration/windows-commands/gpupdate), auf eine Auf [Eingabeaufforderung](https://www.collinsdictionary.com/dictionary/german-english/eingabeaufforderung)  ausgeführt, angewendet.  Es könnte notwendig sein das AD-DM System neu zu starten, damit die Gruppenrichtlinieobjekte vollestandig [abgearbeitet](https://www.collinsdictionary.com/dictionary/german-english/abarbeiten) werden kann.  |
| :---- |

Als Folge diesen **Vorbereitungen** wird es möglich jede**s** System und [Gerät](https://www.collinsdictionary.com/dictionary/german-english/gerat), das Bestandteil der lokale AD Umgebung und [Organisation](https://www.collinsdictionary.com/dictionary/german-english/organisation)s[**einheit**](https://www.collinsdictionary.com/dictionary/german-english/einheit) auf der diese **Gruppenrichtlinie**objekte angewendet ist,  [automatisch](https://www.collinsdictionary.com/dictionary/german-english/automatisch) mit **Intune Endpoint Management (IEM)** registrieren lassen, wenn ein Benutzer auf dem Gerät mit seinem Microsoft 356 Benutzername anmelden sich anmeldet.  

Es gibt verschiedenen Weise zu **feststellen, dass** ein Gerät eigentlich mit **Intune Endpoint Management (IEM)** registriert ist.

1. Nachdem man auf dem Gerät sich anmeldet, wird ein [**Eingabeaufforderung**](https://www.collinsdictionary.com/dictionary/german-english/eingabeaufforderung)  **mit erhöten** Rechten gestarten, dann den Befhel [**dsregcmd /status**](https://learn.microsoft.com/en-us/entra/identity/devices/troubleshoot-device-dsregcmd) ausgeführt. Die Ausgabe des **vorherig**en Behfels soll nun **enthalten** die folgende [**Einträge**](https://www.collinsdictionary.com/dictionary/german-english/eintrag):  
     
   \- MDM terms of use URL	: deine **IEM** terms of use URL  
   \- MDM discovery URL		: deine **IEM** discovery URL  
   \- MDM compliance URL 	: deine **IEM** compliance URL  
     
   Man findet diesen URLs in **Azure Active Directory Admin Center** bzw. **Microsoft Entra ID** unter **Mobility (MDM and MAM) \>** Microsoft Intune.   
      
2. **Es läßt sich** direkt im **Intune Endpoint Management (IEM)** überprüfen, ob das Gerät registriert worden ist, dazu navigiert man auf **All Devices** auf das IEM Portal.
