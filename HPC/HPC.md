**Why is Azure HPC Cache retired?**

Azure HPC Cache is being retired ON September 30, 2025 because the underlying Cloud Service (classic) deployment model, which it relies on for Bursting to Cloud Services, is being retired. 

Microsoft encourages users to transition to newer options like:

Azure HPC Cache brings the scalability of cloud computing to your existing workflow. This service can be used even for workflows where your data is stored across WAN links, such as in your local datacenter network-attached storage (NAS) environment.

# HPC Orchestrators & Workload Managers

### Are Slurm and Apache Mesos comparable products?

**No, Slurm and Apache Mesos are not directly comparable products** **because they serve different primary purposes**, **though both are resource management systems**. 

* **Slurm is a workload manager** designed specifically for HPC environments, excelling at batch job scheduling and resource allocation on supercomputers.   
    
* In contrast, **Mesos is a cluster manager** that focuses on abstracting cluster resources to run diverse distributed applications, such as Hadoop, Spark, and Docker containers, across cloud infrastructures and data centres.

### What are the differences between an HPC orchestrator and an HPC workload manager?

**An HPC workload manager** (e.g., Slurm, Torque) focuses on managing and scheduling jobs within a traditional, fixed HPC cluster, optimising resource use, managing queues, and enforcing policies. 

**An HPC orchestrator** **is a higher-level system**, **often cloud-based**, that **automates the deployment, configuration, and scaling of entire HPC environments,** **including the underlying infrastructure, software, and storage, and can integrate with or utilise workload managers to execute tasks**.
